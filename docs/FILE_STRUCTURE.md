# File structure

This page maps the current repository layout and the purpose of each top-level path.

## Top-level paths
- `AGENTS.md`: agent workflow rules for this repository.
- `README.md`: project overview and quick-start entry point.
- `LICENSE`: repository license terms.
- `pip_requirements.txt`: runtime Python dependencies.
- `pip_requirements-dev.txt`: developer and test dependencies.
- `pyproject.toml`: package metadata and build configuration.
- `source_me.sh`: shell bootstrap for local command execution.

## Code directories
- `screenshot/`: runtime package modules.
- `tests/`: pytest suites and lint/compliance helper scripts.
- `devel/`: development utilities.
- `docs/`: documentation set for usage, standards, and repo policies.

## Generated artifacts
- `report_ascii_compliance.txt`, `report_bandit.txt`, `report_import_dot.txt`, `report_import_requirements.txt`: test/lint reports generated by the test suite.
